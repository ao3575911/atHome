# atHome v0.3 Alpha Release Checklist

Use this checklist as the release-truth gate before tagging `v0.3.0-alpha`.

## Package Manager

- [ ] Install from the root with `npm install`.
- [ ] Confirm `package-lock.json` is current and committed.
- [ ] Do not publish pnpm-based setup or run commands for this release line.

## Verification Gates

- [ ] `npm run typecheck`
- [ ] `npm test`
- [ ] `npm run lint`
- [ ] `npm run build:web`
- [ ] `npm audit --audit-level=moderate`

If the audit gate fails and the fix requires a breaking force upgrade, document the accepted risk in `docs/security/npm-audit-v0.3.md` before release.

## Documentation Gates

- [ ] `README.md` quickstart uses npm workspace commands.
- [ ] `docs/cheatsheet.md` copy-paste commands use npm workspace commands.
- [ ] Release notes link the hosted-registry, key-custody, revocation, and audit-log docs.
- [ ] Security notes cover demo private-key export, bootstrap production blocking, mutation authorization, and audit advisories.

## v0.3 Alpha Readiness

- [ ] Hosted registry boundaries are documented.
- [ ] Production key-custody boundaries are documented.
- [ ] Demo fixtures are regenerated or clearly marked as local-only/non-production.
- [ ] Web/API integration path is documented.
- [ ] Known deferred work is tracked in issues or release notes.
