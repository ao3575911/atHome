# atHome v0.3 Alpha Release Checklist

Use this checklist as the release-truth gate before tagging `v0.3.0-alpha`.

## Package Manager

- [x] Install from the root with `npm install`.
- [x] Confirm `package-lock.json` is current and committed.
- [x] Do not publish pnpm-based setup or run commands for this release line.

## Verification Gates

- [x] `npm run typecheck`
- [x] `npm test`
- [x] `npm run lint`
- [x] `npm run build:web`
- [x] `npm audit --audit-level=moderate` — fails with 7 moderate (0 high); accepted risk documented in `docs/security/npm-audit-v0.3.md`

## Hosted Backend Validation (v0.4 Priority Lane)

- [ ] `DATABASE_URL=postgres://... npm run verify:postgres` (requires `pg` available in the local environment)
- [ ] Run GitHub Actions workflow `Verify` via `workflow_dispatch` with `run_hosted_postgres=true` and secret `ATHOME_VERIFY_DATABASE_URL` configured.
- [ ] Record command output in release notes or issue thread when validating hosted-registry durability work.

If the audit gate fails and the fix requires a breaking force upgrade, document the accepted risk in `docs/security/npm-audit-v0.3.md` before release.

## Documentation Gates

- [x] `README.md` quickstart uses npm workspace commands.
- [x] `docs/cheatsheet.md` copy-paste commands use npm workspace commands.
- [x] Release notes link the hosted-registry, key-custody, revocation, and audit-log docs. (README serves as release notes; links `docs/specs/v0.3-hosted-registry-architecture.md`, `docs/key-custody-plan.md`, `docs/distributed-revocation-model.md`, `docs/transparency-and-audit-log.md`.)
- [x] Security notes cover demo private-key export, bootstrap production blocking, mutation authorization, and audit advisories.
